🧪 Cypress Test Automation

Este repositório contém testes automatizados end-to-end (E2E) desenvolvidos com Cypress para validar fluxos importantes de uma aplicação web.

Os testes foram criados como parte dos meus estudos em Quality Assurance e automação de testes. Para a prática dos cenários, utilizei a aplicação WebDojo como ambiente de aprendizado do curso Ninja do Cypress.

O objetivo deste projeto é praticar automação de testes, validar comportamentos da aplicação e estruturar testes reutilizáveis utilizando boas práticas de QA.

🚀 Cenários automatizados

Login
Os testes automatizados validam diferentes cenários de autenticação do usuário, incluindo login com credenciais válidas, login com senha inválida, login com email inválido e validação das mensagens de erro exibidas pela aplicação.

Formulário de Consultoria
Foi automatizado o fluxo completo de envio do formulário de consultoria, incluindo preenchimento dos campos obrigatórios, upload de arquivo, seleção de tecnologias, seleção de tipo de consultoria, marcação de checkboxes e validação de envio com sucesso.
Também foram criados testes negativos para validar o comportamento da aplicação quando campos obrigatórios não são preenchidos.

🛠 Tecnologias utilizadas

Cypress – framework para automação de testes end-to-end
JavaScript – linguagem utilizada na escrita dos testes
Node.js – ambiente de execução do projeto
Git e GitHub – controle de versão e hospedagem do repositório

▶️ Como executar os testes

Para executar o projeto é necessário instalar as dependências utilizando o gerenciador de pacotes Node.
Depois disso, os testes podem ser executados utilizando o Cypress em modo interativo ou diretamente pelo terminal.

📚 Observação

Este projeto foi desenvolvido com fins educacionais para praticar automação de testes com Cypress.
A aplicação WebDojo foi utilizada apenas como sistema sob teste dentro do ambiente de aprendizado do curso Ninja do Cypress.
